<task type="checkpoint:automated">
  <name>Task 1: Start the production stack and run automated checks</name>
  <action>
Run the following automated checks before asking for human verification:

1. Stop any running containers from previous dev sessions:
   ```bash
   docker compose down 2>/dev/null || true
   ```

2. Build and start the production stack:
   ```bash
   docker compose up --build -d
   ```

3. Wait for backend health check to pass (up to 120 seconds):
   ```bash
   for i in $(seq 1 24); do
     STATUS=$(docker compose ps backend --format json | python3 -c "import sys,json; d=json.load(sys.stdin); print(d[0].get('Health', 'unknown'))" 2>/dev/null || echo "unknown")
     echo "Backend health: $STATUS (attempt $i/24)"
     [ "$STATUS" = "healthy" ] && break
     sleep 5
   done
   ```

4. Verify both containers are running:
   ```bash
   docker compose ps
   ```

5. Curl the health endpoint via nginx (verifies proxy works):
   ```bash
   curl -f http://localhost:3000/api/health && echo "Backend reachable via nginx"
   ```

6. Verify the React app HTML is served:
   ```bash
   curl -s http://localhost:3000/ | grep -q "<!DOCTYPE html>" && echo "React app served by nginx"
   ```

7. Validate the compose file:
   ```bash
   docker compose config --quiet && echo "docker-compose.yml valid"
   ```

Report results. If any check fails, diagnose and fix before proceeding to the human checkpoint.

Common issues to look for:
- 502 Bad Gateway: backend not healthy yet (increase start_period or wait longer)
- 404 on /api/health: nginx proxy_pass stripping/adding prefix incorrectly
- Empty HTML response: mime.types not included in nginx.conf
  </action>
  <verify>
    <automated>curl -sf http://localhost:3000/api/health && curl -s http://localhost:3000/ | grep -q "DOCTYPE html" && echo "All automated checks passed"</automated>
  </verify>
  <done>Backend returns {"status": "ok"} via /api/health; React app HTML loads from port 3000; docker compose ps shows both services healthy</done>
</task>
